Step 2: Edge Protection — Detail

How the edge detector protects UI details from the AI mask

Edge Protection Algorithm
Input
Original source image pixels + Raw AI mask alpha channel
1. Compute gradient magnitude per pixel
For each pixel, compute horizontal & vertical gradients across RGB channels:
gx = |R(x+1) - R(x-1)| + |G(x+1) - G(x-1)| + |B(x+1) - B(x-1)|
gy = |R(y+1) - R(y-1)| + |G(y+1) - G(y-1)| + |B(y+1) - B(y-1)|
gradient = sqrt(gx*gx + gy*gy)

Normalize to 0-255 range. This finds borders, text edges, ornate detail.
2. Build edge strength map
Threshold the gradient to create an "edge confidence" value:
edgeStrength[i] = clamp(gradient[i] - minEdge, 0, maxEdge) / maxEdge

minEdge ~30 (ignore noise/subtle texture)
maxEdge ~120 (strong borders get full protection)

Gold scrollwork edges: gradient ~80-200 → strong protection
Cave rock texture: gradient ~10-40 → weak or no protection
Text/icons: gradient ~60-150 → good protection
3. Dilate edge map
Expand strong edges by 2-3px radius so protection covers the border AND its immediate neighbors.
This prevents the AI from eating one side of a thin gold border.
4. Merge with AI mask
Where AI mask says "remove" (alpha=0) but edge strength is high:
finalAlpha[i] = max(aiAlpha[i], edgeStrength[i] * 255)

This "rescues" pixels the AI incorrectly classified as background
but that are clearly part of a UI border or detail.
Output
Edge-protected alpha channel — AI decisions preserved where edges are weak (game world), overridden where edges are strong (UI borders)

Why This Works for Game UI

Game UI elements (gold borders, stone frames, text, icons) have sharp, high-contrast edges by design — they need to be readable over any game scene. The 3D game world behind them has softer, more organic gradients (fog, lighting, rock surfaces). The edge detector exploits this natural difference.
