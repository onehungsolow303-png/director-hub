Step 3: Spill Cleanup — Detail

Remove color contamination from the game world at mask boundaries

Spill Cleanup Algorithm
The Problem
At the boundary between UI and game world, pixels are a blend of both. A gold border pixel next to purple fog contains some purple. If we just set alpha, the purple "spill" shows as a color fringe on transparent backgrounds.
Without Spill Cleanup
Purple fringe visible on gold border edges
With Spill Cleanup
Clean gold edge, no color contamination
1. Identify boundary pixels
Pixels where 0 < alpha < 255 after edge protection.
These are the transition zone between kept UI and removed background.
2. Sample nearby "clean" foreground color
For each boundary pixel, look inward (toward fully opaque neighbors) to find
the "true" UI color without contamination.
cleanColor = average of neighbors where alpha == 255 within 3px radius
3. Unpremultiply + replace
Existing math from the codebase:
// Unpremultiply to remove background contribution
// Then clamp each channel toward the clean foreground color
// Strength scales with how close pixel is to the edge (lower alpha = more spill)

The existing applyAlphaToImage() decontamination logic already does this —
the enhancement is feeding it better background color estimates
from the AI mask boundary rather than the single sampled background color.
4. Per-boundary background estimation
Key enhancement over current approach:

Current: uses one global background color for all decontamination
Enhanced: for each boundary pixel, sample the nearby removed pixels
(alpha == 0 neighbors) to get the local background color.

localBgColor = average of neighbors where alpha == 0 within 4px radius

This means a gold border next to purple fog gets purple removed,
while a stone frame next to dark blue rocks gets blue removed.
Each edge gets the right cleanup for its local context.
Output
Final RGBA image — UI elements with clean transparent edges, no color fringe from the game world regardless of what color the background was locally.
